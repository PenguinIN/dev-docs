SDK Methods

In this section we present the methods availed to the application developer through our SDK followed by best practices to integrate it within a mobile application.


1.1 Initialize

Returns initialization success or failure value.


SDK Configuration

These steps describe the process of initialization:

  • Validate configurations.
  • Validate OS support.
  • Initialize input sources for detection platform.

Save the results and Log for analytics (if supported)


SDK Configuration

Parameters Description Value Type Mandatory Default Value
Server IP API IP address String Yes
Service Name Data API service name String Yes
userName Front End UserID String Yes
Enable Log Enable\Disable SDK log Boolean No False

1.4Trigger Functions

Can be used to define trigger functions on data changes in SDK.


  • didStartPenguinIn ()

    fired when initializing SDK.

    Response

    Value Description Value Type
    result Detected PoIs [PoI]

  • didPointOfInterestChange (result:[PoI])

    Returns list of detected PoIs with trust factor for each.

    Response

    Value Description Value Type
    result Detected PoIs [PoI]

  • didFailInitilize (msg: String)

    fired when failed to initialize SDK trust factor for each.

    Response

    Value Description Value Type
    msg Error message String

  • didCompleteLoading (msg: String)

    fired when complete loading data

    Response

    Value Description Value Type
    msg Return status message String

    SDK status Progress Look Up

    Value Description
    None Not initialized yet
    Initializing Initializing process
    LoadingData Loading beacons and PoIs data for detection process
    EmptyScanFilter SDK doesn’t have any beacon for scan Loading error or empty data
    Detection SDK is ready and detection mode running successfully.
